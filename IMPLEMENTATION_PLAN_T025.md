# T-025: Divergence Report — Reconciliation Engine

## Context

The Telco2 dataset contains two parallel views of the same network:

| Layer | Table | Rows | Represents |
|-------|-------|------|------------|
| **CMDB (Intent)** | [network_entities](file:///Users/himanshu/Projects/Pedkai/backend/app/scripts/load_telco2_tenant.py#170-312) | 784,048 | What the organisation *thinks* the network is |
| **CMDB (Intent)** | [topology_relationships](file:///Users/himanshu/Projects/Pedkai/backend/app/scripts/load_telco2_tenant.py#453-564) | 1,660,086 | Declared relationships |
| **Ground Truth (Reality)** | `gt_network_entities` | 811,064 | What the network *actually is* |
| **Ground Truth (Reality)** | `gt_entity_relationships` | 1,970,387 | Actual relationships |
| **Validation** | [divergence_manifest](file:///Users/himanshu/Projects/Pedkai/backend/app/scripts/load_telco2_tenant.py#1187-1263) | 459,769 | Pre-seeded divergences (for scoring Pedkai's detection accuracy) |

**The pre-seeded [divergence_manifest](file:///Users/himanshu/Projects/Pedkai/backend/app/scripts/load_telco2_tenant.py#1187-1263) was generated by Sleeping-Cell-KPI-Data** to provide ground-truth labels for divergence detection. It should NOT be displayed as the report. Instead, Pedkai must *discover* divergences by comparing CMDB against reality, then score its detection accuracy against [divergence_manifest](file:///Users/himanshu/Projects/Pedkai/backend/app/scripts/load_telco2_tenant.py#1187-1263).

> [!IMPORTANT]
> **How the seeded data is findable:** `gt_network_entities` and `gt_entity_relationships` use the same entity IDs as the CMDB tables (`network_entities.id::text = gt_network_entities.entity_id`). The join keys align perfectly — set-difference operations immediately reveal dark nodes, phantom nodes, dark edges, phantom edges, identity mutations (external_id drift), and attribute divergences (JSONB attribute comparison). This is *too easy* — in a real deployment, the "ground truth" would come from live telemetry, not a parallel table. For the next tenant, consider:
> - Not pre-loading ground truth tables at all, forcing Pedkai to derive reality from `kpi_metrics` telemetry
> - Using different entity ID schemes between CMDB and telemetry so identity resolution is required
> - Placing reality data only in time-series metrics tables, not in a parallel entity table

## Proposed Changes

### Backend: Reconciliation Engine

#### [NEW] [reconciliation_engine.py](file:///Users/himanshu/Projects/Pedkai/backend/app/services/reconciliation_engine.py)

Core reconciliation service. Compares CMDB state against ground truth to detect divergences algorithmically.

**Algorithm (all SQL, no ORM load):**

```
Confirmed_Nodes = CMDB_Entities ∩ GT_Entities         (JOIN on id)
Phantom_Nodes   = CMDB_Entities - GT_Entities          (LEFT JOIN, GT IS NULL)
Dark_Nodes      = GT_Entities - CMDB_Entities          (LEFT JOIN, CMDB IS NULL)

For Confirmed_Nodes:
  Identity_Mutations = WHERE cmdb.external_id ≠ gt.external_id
  Dark_Attributes    = WHERE cmdb.attributes->key ≠ gt.attributes->key
                       (compare vendor, band, sla_tier, etc.)

Confirmed_Edges = CMDB_Edges ∩ GT_Edges               (JOIN on from/to/type)
Phantom_Edges   = CMDB_Edges - GT_Edges
Dark_Edges      = GT_Edges - CMDB_Edges
```

**Scoring against [divergence_manifest](file:///Users/himanshu/Projects/Pedkai/backend/app/scripts/load_telco2_tenant.py#1187-1263):** After detection, compare discovered divergences with pre-seeded manifest. Calculate precision, recall, and F1 per divergence type.

**Key methods:**

| Method | Description |
|--------|-------------|
| `run(tenant_id)` | Execute full reconciliation, return `ReconciliationReport` |
| `_score_against_manifest(tenant_id)` | Compare engine's findings vs pre-seeded ground truth |
