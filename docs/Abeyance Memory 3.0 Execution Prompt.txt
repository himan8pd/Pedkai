Abeyance Memory v2.0 is a competent correlation engine only.

ranking, threshold-based snap detection, provenance tracking.

Gap concentrated in six root causes:

  2. Embedding architecture is economically catastrophic at scale.

EMBEDDING ARCHITECTURE REDESIGN (MANDATORY)

────────────────────────────────────────────────────────────────────────────────

The current concatenated 4x384=1536 single-vector architecture is replaced.

All designs in this reconstruction MUST use the following storage and scoring

model.

Embedding Model:

  - NetoAI T-VEC 1.5B (telecom-specific, MIT license, 1536-dim output,

    0.938 on telecom triplet benchmark, 0.825 MTEB)

    HuggingFace: NetoAISolutions/T-VEC

    Base: Alibaba-NLP/gte-Qwen2-1.5B-instruct

    Runs on CPU (1.5B params, ~3GB RAM). No GPU required for embeddings.

Text Generation Model (entity extraction, hypothesis generation):

  - NetoAI TSLAM-8B (telecom-specific, fine-tuned from Llama-3.1-8B,

    86.2% on telecom domain benchmark, 300-500ms inference on GPU)

  - Fallback for no-GPU deployments: TSLAM-4B (edge-optimised)

Deployment Model:

  - Both models run locally in the client's environment.

  - No cloud LLM dependency. Zero marginal cost per call.

  - External LLM calls occur only on explicit client request, paid by client,

    behind client-controlled cost ceilings.

  - Data sovereignty: telco network data never leaves the client boundary.

Storage Schema (replaces the single enriched_embedding Vector(1536) column):

  emb_semantic      Vector(1536)  -- T-VEC embed of content + extracted entities

  emb_topological   Vector(1536)  -- T-VEC embed of Shadow Topology neighbourhood

  emb_temporal      Vector(256)   -- sinusoidal encoding (no LLM, pure math)

  emb_operational   Vector(1536)  -- T-VEC embed of failure modes + fingerprint

  mask_semantic     Boolean NOT NULL DEFAULT FALSE

  mask_topological  Boolean NOT NULL DEFAULT FALSE

  mask_operational  Boolean NOT NULL DEFAULT FALSE

  -- temporal is always valid; no mask needed

Drop: enriched_embedding Vector(1536), embedding_mask JSONB.

No concatenated vector exists in the redesigned system.

Enrichment Chain (per fragment):

  1. T-VEC call: embed content + entities       -> emb_semantic (1536)

  2. T-VEC call: embed neighbourhood description -> emb_topological (1536)

  3. Math: sinusoidal time-of-day/day-of-week    -> emb_temporal (256)

  4. T-VEC call: embed failure mode + fingerprint -> emb_operational (1536)

  If any T-VEC call fails: column stays NULL, mask stays FALSE.

  No zero-filling. No hash embedding fallback. NULL means absent.

Snap Scoring (replaces single cosine over concatenated vector):

  - Compute cosine similarity independently per dimension (only where both

    fragments have mask=TRUE for that dimension).

  - Temporal similarity: cosine over 256-dim sinusoidal vectors (always available).

  - Entity overlap: Jaccard over entity sets (always available, structural).

  - Combine per-dimension scores using weight profiles.

  - When a dimension is unavailable (either mask FALSE), redistribute its

    weight proportionally across available dimensions. Never score against NULL.

  - This means the embedding mask is consumed at scoring time, not stored and

    ignored. Audit finding 6.2 is resolved architecturally.

Cold Storage:

  - Same four-column schema for ColdFragmentORM.

  - IVFFlat index per T-VEC column (3 indexes: semantic, topological, operational).

  - Cold search queries emb_semantic by default; multi-index fusion is optional.

  - IVFFlat list count must scale with sqrt(n), not fixed at 100.

Migration:

  - Existing concatenated embeddings cannot be decomposed (L2 normalisation

    destroyed sub-vector boundaries). Existing fragments get mask_*=FALSE on

    new columns and decay naturally.

  - New fragments use new schema exclusively.

  - Drop old columns once no old-schema fragments remain active.

DISCOVERY MECHANISM SPECIFICATIONS

────────────────────────────────────────────────────────────────────────────────

1. Surprise Metrics

   Score: -log(P(observation | current model)).

   "Current model" = rolling distribution of snap scores per failure mode

   profile per tenant. No new model needed -- track score distributions and

   flag statistical outliers.

   High surprise triggers escalation to discovery evaluation.

   Also serves as anomalous snap rate detection (addresses audit finding 7.2).

2. Ignorance Mapping

   Surface sparse, unstable, or low-confidence regions of the evidence space.

   Implementation: track entity extraction success rates per entity type,

   per source type, per time window. Track embedding mask distributions

   (the mask data is already stored -- this mechanism reads what the system

   already writes). Identify blind spots where fragments decay to zero

   without ever being evaluated (audit finding 4.2).

   Direct exploration effort toward high-ignorance regions.

3. Negative Evidence / Disconfirmation

   The system currently cannot anti-corroborate. Decay is time-based, not

   evidence-based (audit finding 4.4).

   Implement: operator-driven reclassification of ACTIVE fragments as

   "investigated, not relevant." Accelerated decay for disconfirmed fragments.

   Propagate disconfirmation to similar fragments (reduce snap scores for

   future fragments matching a disconfirmed pattern).

   API endpoint for operator-driven negative evidence on ACTIVE fragments

   (not just SNAPPED, which INV-5 already covers).

4. Cross-Cluster Insight Detection

   Detect bridge nodes (articulation points) and high-betweenness fragments

   in the accumulation graph. Pure graph algorithm on existing structure.

   No LLM calls. No new data.

   Bridge fragments that connect otherwise-disconnected clusters are

   structural discoveries invisible to pairwise snap scoring.

5. Outcome-Linked Scoring Calibration

   Connect operator resolution actions (acknowledge, dismiss, escalate,

   resolve) on SNAPPED fragments back to the snap scores and weight profiles

   that produced them.

   Build a feedback table: snap_decision_record_id -> operator_action ->

   outcome (true positive, false positive, missed).

   Use this data to empirically validate and adjust weight profiles.

   This is the prerequisite for resolving audit finding 2.4 (arbitrary weights)

   with data instead of hand-tuning.

6. Pattern Conflict Detection

   Identify contradictions: two fragments with high entity overlap but

   opposite polarity (e.g., "link up" vs "link down" for same entity within

   a time window). Flag these as conflicts requiring investigation.

   Do NOT attempt automated "hidden variable search" -- that is scope creep

   into unsolved causal inference. Surface the conflict; let operators and

   higher-tier mechanisms investigate.

7. Temporal Sequence Modelling

   For each entity, maintain an ordered log of fragment types/states.

   Build transition probability matrices (simple counting, no LLM).

   This is cheap infrastructure that enables expectation violation detection

   and causal direction testing at higher tiers.

8. Hypothesis Generation

   Convert recurring snap patterns into testable claims using TSLAM-8B.

   Input: snap pair (two fragments, their entities, their per-dimension

   scores). Output: a falsifiable claim, conditions for confirmation,

   conditions for refutation. Track confirmations and refutations over time.

   Replace the current hypothesis_id UUID label with a real hypothesis object.

9. Expectation Violation Detection

   Compare observed entity state transitions against the transition

   probability matrices from #7. Flag deviations where observed sequence

   diverges from expected (e.g., A->B->C expected, A->B->D observed).

   Violation severity = surprise of the unexpected transition.

10. Causal Direction Testing

    Granger-style temporal precedence: if fragments about entity A

    consistently precede fragments about entity B by a stable lag, flag as

    directional causal candidate. Computable from timestamps + entity data.

    Cheaper than full counterfactual simulation. Not proof of causation,

    but stronger than symmetric similarity.

11. Pattern Compression Discovery

    Define a pattern grammar (prerequisite). Collapse multiple snap patterns

    into simpler rules. Compression gain = discovery signal (a regularity

    that was not explicitly programmed).

12. Counterfactual Simulation

    Remove candidate events from historical sequences and re-run snap

    scoring to measure causal impact. O(n) replays per candidate.

    Batch job only. Run during maintenance windows, not real-time.

    Only meaningful once the scoring pipeline is end-to-end functional

    (topology wired, masks consumed, weights calibrated).

13. Meta-Memory

    Track which search areas (entity types, failure modes, time windows,

    topological regions) have historically produced confirmed hypotheses

    vs unproductive noise. Bias future exploration toward productive areas.

    Requires outcome tracking (#5) to distinguish signal from volume.

14. Evolutionary Pattern Discovery

    Treat confirmed patterns as evolving entities with fitness metrics:

    predictive power (from outcome tracking), stability (from temporal

    persistence), novelty (from surprise metrics), compression (from

    pattern compression). Select, mutate, and recombine patterns.

    This is the most complex mechanism and requires all preceding tiers

    to be operational and calibrated.

ARCHITECTURAL CONSTRAINTS

────────────────────────────────────────────────────────────────────────────────

Forbidden: arbitrary thresholds without empirical derivation methodology,

without documented basis, floating-point instability, hash embedding fallbacks,

zero-filled sub-vectors passed off as valid embeddings.

All algorithms must be deterministic, bounded, explainable, and reproducible.

run_in_executor for T-VEC on CPU, vLLM/TGI for TSLAM-8B). The async context

of the FastAPI runtime (loop.is_running() == True) is a given. Design

accordingly -- no blocking calls, no fallback to hash embeddings.
