Controlled Telemetry Replay & Streaming Intelligence Pipeline

Objective

Build a controlled telemetry replay pipeline that simulates real-time production traffic by streaming historical Parquet telemetry data into Apache Kafka, allowing backend services to process the data into PostgreSQL and TimescaleDB while dashboards visualize system learning and memory evolution.

The system must behave like a live production environment while operating within constrained infrastructure.

⸻

Live-Stream Compatibility Requirement (Architectural Constraint)

The telemetry replay mechanism must be engineered as a drop-in stand-in for real production telemetry ingestion.

Core Principle

Downstream systems must remain completely unaware that telemetry originates from historical Parquet data rather than a live network.

Future Transition Requirement

At any time, the replay source must be replaceable with:

Live telemetry streams from a client production network

with zero required changes to:
	•	Kafka infrastructure
	•	Topic design
	•	Backend consumers
	•	Database schemas
	•	Inference systems
	•	Dashboards
	•	Abeyance Memory systems
	•	Intelligence and analytics layers

Design Requirements

The replay service must therefore:
	•	Emit messages identical to live production telemetry
	•	Preserve production-grade event cadence and sequencing
	•	Use identical Kafka topic structures
	•	Use identical message schemas
	•	Match delivery semantics of real streaming sources

Abstraction Model

Telemetry Source (Replaceable Layer)
    ├── Historical Parquet Replay (current mode)
    └── Live Client Network Stream (future mode)
            ↓
        Kafka Ingestion Layer (permanent)
            ↓
        Processing & Intelligence Platform

Strict Constraint

Parquet-based replay is only a temporary telemetry source, not a platform dependency.
No downstream component may contain replay-specific logic.

The platform must treat replayed telemetry as production-equivalent data.

Implementation Blueprint

Step 1 — Parquet → Kafka Streaming

Python Skeleton

import pyarrow.parquet as pq
import time
from confluent_kafka import Producer
import json

table = pq.read_table("telemetry.parquet")
df = table.to_pandas().sort_values("timestamp")

producer = Producer({'bootstrap.servers': 'localhost:9092'})

ACCELERATION = 120

prev_ts = None

for _, row in df.iterrows():
    if prev_ts:
        delta = (row.timestamp - prev_ts).total_seconds()
        time.sleep(delta / ACCELERATION)

    topic = f"telemetry.{row.domain}.{row.metric_type}"

    producer.produce(topic, json.dumps(row.to_dict()).encode())
    producer.poll(0)

    prev_ts = row.timestamp
