EMBEDDING ARCHITECTURE (all agents must assume this)

Embedding model: NetoAI T-VEC 1.5B
  - Telecom-specific, MIT license, 1536-dim output
  - HuggingFace: NetoAISolutions/T-VEC
  - Runs on CPU (~3GB RAM). No GPU required for embeddings.

Text generation model: NetoAI TSLAM-8B
  - Telecom-specific, fine-tuned from Llama-3.1-8B
  - For entity extraction and hypothesis generation
  - Fallback: TSLAM-4B for edge/CPU-only deployments

Deployment model:
  - Both models run locally in client environment
  - No cloud LLM dependency. Zero marginal cost per call.
  - External LLM only on explicit client request, paid by client.

Storage schema (replaces single enriched_embedding Vector(1536)):
  emb_semantic      Vector(1536)  -- T-VEC embed of content + entities
  emb_topological   Vector(1536)  -- T-VEC embed of Shadow Topology neighbourhood
  emb_temporal      Vector(256)   -- sinusoidal encoding (no LLM, pure math)
  emb_operational   Vector(1536)  -- T-VEC embed of failure modes + fingerprint
  mask_semantic     Boolean NOT NULL DEFAULT FALSE
  mask_topological  Boolean NOT NULL DEFAULT FALSE
  mask_operational  Boolean NOT NULL DEFAULT FALSE

Rules:
  - If T-VEC call fails: column is NULL, mask is FALSE. No zero-fill. No hash fallback.
  - Snap scoring: cosine per dimension independently, mask-aware weight redistribution.
  - Temporal embedding is always valid (pure math, no model dependency).
